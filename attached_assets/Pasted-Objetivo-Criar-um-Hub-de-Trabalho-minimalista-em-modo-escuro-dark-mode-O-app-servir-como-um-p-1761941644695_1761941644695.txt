Objetivo: Criar um "Hub de Trabalho" minimalista em modo escuro (dark mode). O app servirá como um painel central para a equipe, mostrando quem está online, permitindo acesso rápido a uma sala de reunião principal do Google Meet e integrando recursos do Google Drive.Pilha de Tecnologia (Sugestão para Replit):Frontend: React (Vite)Estilo: TailwindCSSBackend & DB: Firebase (para Autenticação Google e Banco de Dados Firestore em tempo real)Tema e Estilo:Modo: Dark Mode por padrão. Fundo principal (bg-gray-900), fundos de card/painel (bg-gray-800), texto principal (text-gray-100), texto secundário (text-gray-400).Estética: Minimalista, cantos arredondados (rounded-lg), sombras sutis (shadow-md).Ícones: Usar lucide-react (ou similar). Absolutamente sem emojis.Estrutura de Páginas e Componentes:Página 1: Login (Rota: /login)Layout centralizado.Título: "Hub da Equipe" (ou nome do projeto).Um único botão: "[Ícone de Google] Entrar com Google".Lógica: Usa a autenticação do Firebase (Google Provider). Após o login, redireciona para o Dashboard.Página 2: Dashboard (Rota: /) - Página PrincipalLayout: Uma barra lateral esquerda fixa (para navegação/recursos) e uma área de conteúdo principal (para os cards da equipe).Proteção: Esta rota deve ser protegida; redireciona para /login se o usuário não estiver autenticado.Componente 1: Barra Lateral (Sidebar)Largura fixa (aprox. w-72). Fundo bg-gray-800.Seção 1: Perfil do Usuário (Topo)Foto do Google, Nome do Usuário, Email.Botão "Sair" [Ícone LogOut].Seção 2: Sala de Reunião (Principal)Título: "Sala Principal".Um grande botão de call-to-action: "[Ícone Video] Entrar na Chamada".Lógica:Este botão aponta para um link fixo do Google Meet (que será configurado no backend ou em variáveis de ambiente, ex: MEET_URL=httpsa://meet.google.com/lookup/nome-da-sala).Abre o link em uma target="_blank".Ao clicar, atualiza o status do usuário no Firestore para "Em Chamada".Seção 3: Google Drive (Integração)Título: "Documentos da Equipe".Botão: "[Ícone Folder] Abrir Drive".Lógica: Este botão deve usar a API do Google Picker para abrir um modal "pop-up" que permite ao usuário navegar e selecionar arquivos do Google Drive da equipe.Seção 4: Google Calendar (Integração Opcional)Título: "Agenda da Equipe".Um iframe embutido de uma agenda pública/compartilhada específica do Google Calendar da equipe.Componente 2: Área de Conteúdo (Grid da Equipe)Fundo bg-gray-900.Título: "Equipe Online".Layout: Um grid responsivo (ex: grid grid-cols-2 md:grid-cols-4 lg:grid-cols-6 gap-4).Componente 3: Card de Membro (MemberCard)Fundo bg-gray-800, rounded-lg, shadow-md, padding-4.Indicador de Status: Um pequeno círculo (badge) no canto superior.Verde (bg-green-500): "Online"Azul (bg-blue-500): "Em Chamada"Cinza (bg-gray-500): "Offline"Foto do Google (circular).Nome do Usuário (font-semibold text-gray-100).Email ou Cargo (text-sm text-gray-400).Lógica de Backend (Firebase/Firestore):Autenticação: Configurar o Firebase Auth para aceitar o Google como provedor.Banco de Dados (Firestore):Coleção: users.Documento: userId (o ID do Google Auth).Estrutura do Documento:{
  "uid": "...",
  "name": "Nome do Usuário",
  "email": "email@dominio.com",
  "photoURL": "url_da_foto.jpg",
  "status": "Online" // Pode ser "Online", "Offline", "Em Chamada"
}
Gerenciamento de Presença (Real-time):O app deve usar onSnapshot (listener do Firestore) na coleção users para exibir o status de todos em tempo real no Grid da Equipe.Ao carregar o app (Login): O app atualiza o documento do usuário logado no Firestore com status: "Online".Ao fechar o app (Browser close/tab close): O app deve tentar usar o evento beforeunload ou onDisconnect do Firebase Realtime Database (se preferir) para atualizar o status para "Offline".Ao clicar em "Entrar na Chamada": Atualiza o status para "Em Chamada".Ao sair da chamada (Difícil de detectar): O status permanecerá "Em Chamada" até que o usuário feche o app ou volte e mude seu status manualmente (para simplificar).
